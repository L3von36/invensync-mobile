col-md-9 col-xl-8 ps-md-5\" role=\"main\">\n            <div class=\"td-content\">\n              <h1>Web</h1>\n              <div class=\"lead\">Drift support in Flutter and Dart web apps.</div>\n              <header class=\"article-meta\"></header>\n              <p>\n                Using modern browser APIs such as WebAssembly and the Origin-Private File System API,\n                you can use drift databases for the web version of your Flutter and Dart applications.\n                Just like the core drift APIs, web support is platform-agnostic:\n                Drift web supports Flutter Web, jaspr, plain <code>package:web/</code> or any other Dart web framework.\n              </p>\n              <p>\n                While an official sqlite3 build for the web exists, it is fairly large and doesn't support most browsers.\n                Drift uses a custom sqlite3 build with a Dart interface sharing a lot of code with the existing native\n                platforms, which enables a fast implementation that works on more browsers.\n              </p>\n              <h2 id=\"supported-browsers\" anchor=\"true\"><span>Supported browsers</span><a href=\"/platforms/web#supported-browsers\">#</a></h2>\n              <p>\n                Drift uses the FileSystem Access API to store databases if it's available. Otherwise, it will fall back to\n                a slower implementation based on IndexedDb in a shared worker. This makes drift available on all modern browsers,\n                even ones that don't support the official sqlite3 build for the web.\n                In some browsers, you need to serve your app with <a href=\"#additional-headers\">additional COOP/COEP headers</a>\n                 for full support (but drift works without that too - the official sqlite3 build doesn't).\n              </p>\n              <table>\n                <thead>\n                  <tr>\n                    <th>Supported Browser</th>\n                    <th>With <a href=\"#additional-headers\">security headers</a></th>\n                    <th>Without security headers</th>\n                  </tr>\n                </thead>\n                <tbody>\n                  <tr><td>Firefox <em>(tested version 114)</em></td><td>Full</td><td>Full</td></tr>\n                  <tr><td>Chrome <em>(tested version 114)</em></td><td>Full</td><td>Good (slightly slower)</td></tr>\n                  <tr>\n                    <td>Chrome on Android <em>(tested version 114)</em></td>\n                    <td>Full</td>\n                    <td>Limited (not with multiple tabs)</td>\n                  </tr>\n                  <tr>\n                    <td>Safari (<em>tested version 16.2</em>)</td>\n                    <td>Good (slightly slower)</td>\n                    <td>Good (slightly slower)</td>\n                  </tr>\n                  <tr><td>Safari Technology Preview <em>(tested 172 (17.0))</em></td><td>Full</td><td>Good</td></tr>\n                </tbody>\n              </table>\n              <p>\n                Firefox currently doesn't support the FileSystem Access API in private browsing windows\n                (IndexedDB is supported from version 115). So drift will fall back to an IndexedDb-based\n                or an in-memory database in private tabs.\n              </p>\n              <p>\n                In Chrome on Android, shared workers aren't supported. So if the <a href=\"#additional-headers\">headers</a>\n                 required\n                for the preferred API are missing, there unfortunately is no way to prevent data races between tabs,\n                which can lead to persistence issues. Drift informs you about the chosen storage mode, so depending on\n                how critical persistence is to your app, you can instruct users to download a native app or use a different\n                browser in that case.\n              </p>\n              <div class=\"admonition note\">\n                <p class=\"admonition-title\">Compatibility check</p>\n                <p>\n                  This page includes a tiny drift database compiled to JavaScript.\n                  You can use it to verify drift works in the browsers you want to target.\n                  Clicking on the button will start a feature detection run, so you can see which file system\n                  implementation drift would pick on this browser and which web APIs are missing.\n                </p>\n                <!--@web_compatibility--><div><button class=\"compat\">Check compatibility</button><pre>Compatibility check not started yet.</pre></div><!--/@web_compatibility-->\n                <p>\n                  More information about these results is available <a href=\"#supported-storage-implementations\">below</a>.\n                </p>\n              </div>\n              <h2 id=\"getting-started\" anchor=\"true\"><span>Getting started</span><a href=\"/platforms/web#getting-started\">#</a></h2>\n              <h3 id=\"prerequisites\" anchor=\"true\"><span>Prerequisites</span><a href=\"/platforms/web#prerequisites\">#</a></h3>\n              <p>\n                On all platforms, drift requires access to <a href=\"https://sqlite.org/\">sqlite3</a>, the popular\n                database system written as a C library.\n                On native platforms, drift can bundle the sqlite3 library with your app or use a copy included on\n                most operating systems.\n              </p>\n              <p>\n                Web browsers don't have builtin access to the sqlite3 library, so it needs to be included with your app.\n                The <code>sqlite3</code> Dart package (used by drift internally) contains a toolchain to compile sqlite3 to WebAssembly\n                so that it can be used in browsers. You can grab a prebuilt <code>sqlite3.wasm</code>\n                 file from <a href=\"https://github.com/simolus3/sqlite3.dart/releases\">its releases page</a>,\n                or <a href=\"#compilation\">compile it yourself</a>.\n                This file needs to be put into the <code>web/</code> directory of your app.\n              </p>\n              <p>\n                Drift on the web also requires you to include a portion of drift as a web worker. This worker will be used to\n                host your database in a background thread, improving performance of your website. In some \n                <a href=\"#supported-storage-implementations\">storage implementations</a>,\n                the worker is also responsible for sharing your database between different tabs in real-time.\n                Again, you can <a href=\"#compilation\">compile this worker yourself</a> or <a href=\"https://github.com/simolus3/drift/releases\">grab one from drift releases</a>.\n              </p>\n              <div class=\"admonition danger\">\n                <p class=\"admonition-title\">Download the correct versions</p>\n                <p>\n                  You should download the version of <code>sqlite3.wasm</code> that matches with the version of <code>sqlite3</code>\n                   you have installed\n                  and download the version of <code>drift_worker.dart.js</code> that matches with the version of \n                  <code>drift</code> you have installed.\n                </p>\n                <p>\n                  To find out which versions of <code>sqlite3</code> and <code>drift</code> you're currently using, check your \n                  <code>pubspec.lock</code> file!\n                </p>\n              </div>\n              <p>In the end, your <code>web/</code> directory may look like this:</p>\n              <div class=\"code-block\">\n                <!--@jaspr_content_snippets:client--><button><svg fill=\"none\" stroke=\"currentColor\" stroke-width=\"2\" stroke-linecap=\"round\" stroke-linejoin=\"round\" viewBox=\"0 0 24 24\" width=\"18px\" height=\"18px\"><rect rx=\"2\" ry=\"2\" x=\"8\" y=\"8\" width=\"14\" height=\"14\"></rect><path d=\"M4 16c-1.1 0-2-.9-2-2V4c0-1.1.9-2 2-2h10c1.1 0 2 .9 2 2\"></path></svg></button><!--/@jaspr_content_snippets:client-->\n                <pre><code><span>web/\n├── favicon.png\n├── index.html\n├── manifest.json\n├── drift_worker.dart.js\n└── sqlite3.wasm</span></code></pre>\n              </div>\n              <div class=\"admonition info\">\n                <p class=\"admonition-title\">Serving wasm files</p>\n                <p>\n                  For browsers to accept the <code>sqlite3.wasm</code> file, it must be served with\n                  <code>Content-Type: application/wasm</code>.\n                  <code>flutter run</code> does this by default, but some webservers might not. After\n                  deploying your app to the web, check that your server is configured to send the\n                  correct <code>Content-Type</code> header for wasm files.\n                </p>\n              </div>\n              <h4 id=\"additional-headers\">Additional headers</h4>\n              <p>\n                On browsers that support it, drift uses the origin-private part of the <a href=\"https://developer.mozilla.org/en-US/docs/Web/API/File_System_Access_API\">FileSystem Access API</a>\n                 to store databases efficiently.\n                As parts of that API are asynchronous, and since sqlite3 expectes a synchronous file system, we need to\n                use two workers with shared memory and <code>Atomics.wait</code>/<code>notify</code>.\n                Just like the official sqlite3 port to the web, <strong>this requires your website to be served with two special headers</strong>:\n              </p>\n              <ul>\n                <li><code>Cross-Origin-Opener-Policy</code>: Needs to be set to <code>same-origin</code>.</li>\n                <li>\n                  <code>Cross-Origin-Embedder-Policy</code>: Needs to be set to <code>require-corp</code> or <code>credentialless</code>.\n                </li>\n              </ul>\n              <p>\n                For more details, see the <a run</code> just like always Drift will fall back to a (slightly slower) implementation (see \n                <a href=\"#supported-storage-implementations\">storages</a>).\n                We While allowed by the web standards, this is only implemented by Firefox.\n                </li>\n                <li>\n                  <code>opfsLocks</code>: Uses the origin-private filesystem access API, but without shared workers.\n                  This requires the <a href=\"#additional-headers\">COOP and COEP headers</a>.\n                </li>\n                <li>\n                  <code>sharedIndexedDb</code>: Uses IndexedDB to store chunks of data, the database is hosted in a shared web worker.\n                </li>\n                <li>\n                  <code>unsafeIndexedDb</code>: This also uses the IndexedDB database, but without a shared worker or another\n                  means of synchronization between tabs. In this mode, <strong>it is not safe for multiple tabs of your app\n                    to access the same database</strong>.\n                  While this was the default mode used by earlier implementations of drift for the web, we now try to\n                  avoid it and it will not be used on modern browsers.\n                </li>\n                <li>\n                  <code>inMemory</code>: When no persistence API is available, drift will fall back to an in-memory database.\n                </li>\n              </ol>\n              <p>\n                The missing browser APIs contributing to a specific implementation being chosen are available in\n                <code>WasmDatabaseResult.missingFeatures</code>.\n                There's nothing your app or drift could do about them. If persistence is important to your app\n                and drift has chosen the <code>unsafeIndexedDb</code> or the <code>inMemory</code> implementation due to a lack of proper\n                persistence support, you may want to show a warning to the user explaining that they have to upgrade\n                their browser.\n              </p>\n              <h3 id=\"customizing-how-databases-are-opened\" anchor=\"true\">\n                <span>Customizing how databases are opened</span><a href=\"/platforms/web#customizing-how-databases-are-opened\">#</a>\n              </h3>\n              <p>\n                To customize which implementation drift uses, you can use the <code>WasmDatabase.probe</code> API.
